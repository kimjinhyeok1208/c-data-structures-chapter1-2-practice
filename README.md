## 🧩 문제 구성

### ✅ 문제 1. 알고리즘 시간 복잡도 비교

#### 🔹 문제 설명

입력 크기 n에 따라 서로 다른 시간 복잡도를 가지는 알고리즘을 구현하고,
각 알고리즘의 **반복 횟수(연산 횟수)**를 카운트하여 비교한다.

#### 🔹 구현 알고리즘

* 선형 시간 알고리즘: O(n)
* 이차 시간 알고리즘: O(n²)
* 로그 시간 알고리즘: O(log n)

#### 🔹 시간 복잡도 분석

| 함수          | 시간 복잡도   | 설명               |
| ----------- | -------- | ---------------- |
| linear      | O(n)     | 입력 크기에 비례하여 반복   |
| quadratic   | O(n²)    | 중첩 반복문 사용        |
| logarithmic | O(log n) | 반복마다 입력 크기 절반 감소 |

---

### ✅ 문제 2. 구조체 기반 학생 성적 관리

#### 🔹 문제 설명

구조체를 이용하여 학생 정보를 저장하고,
학생들의 평균 점수를 계산한 후 **전체 평균 이상인 학생만 출력**한다.

#### 🔹 사용 개념

* 구조체(struct)
* 구조체 배열
* 포인터를 이용한 함수 호출

#### 🔹 시간 복잡도

* 학생 입력: O(n)
* 평균 계산: O(n)
* 평균 이상 학생 출력: O(n)

➡️ 전체 시간 복잡도: **O(n)**

---

### ✅ 문제 3. 재귀 vs 반복 (팩토리얼)

#### 🔹 문제 설명

팩토리얼을 **재귀 함수**와 **반복문**으로 각각 구현하고,
각 방식의 **함수 호출 횟수 및 반복 횟수**를 카운트하여 비교한다.

#### 🔹 시간 복잡도

| 구현 방식 | 시간 복잡도 |
| ----- | ------ |
| 재귀    | O(n)   |
| 반복    | O(n)   |

#### 🔹 공간 복잡도

| 구현 방식 | 공간 복잡도 | 이유           |
| ----- | ------ | ------------ |
| 재귀    | O(n)   | 함수 호출 스택 사용  |
| 반복    | O(1)   | 추가 메모리 사용 없음 |

#### 🔹 결론

> 두 방식은 동일한 시간 복잡도를 가지지만,
> 재귀 방식은 호출 스택으로 인해 추가적인 메모리를 사용한다.

## 🎯 학습 정리

* 시간 복잡도는 **실행 시간**이 아니라 **입력 크기에 따른 증가율**이다.
* 구조체와 포인터를 사용하면 복잡한 데이터를 효율적으로 관리할 수 있다.
* 재귀는 직관적이지만 메모리 사용에 주의해야 한다.
* 반복문은 메모리 측면에서 더 효율적이다.
